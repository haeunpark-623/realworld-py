## 2. 화면 상세

### S-01: Home (게시글 목록)

- **목적**: 비회원·가입 사용자 모두에게 최신 게시글 카드 목록 + 페이지네이션 제공
- **상태**:
  - **default**: 게시글 카드 20개 + 페이지네이션
  - **loading**: 스피너 (API 응답 대기)
  - **empty**: "아직 게시글이 없습니다" 안내
  - **error**: "잠시 후 다시 시도해 주세요" + 재시도 버튼
- **F-ID 매핑**: F-02 (R-F-04)
- **주요 액션**: 글 카드 클릭 → S-02 / 페이지 번호 클릭 → 페이지 이동
- **와이어프레임 (간략)**:
  ```
  ┌─────────────────────────────────────────────────────┐
  │ Header (공통)                                        │
  ├─────────────────────────────────────────────────────┤
  │ <h1>최근 게시글</h1>                                  │
  │ ┌─────────────────────────────────────────────────┐ │
  │ │ [Author username] · 2026-05-21                  │ │
  │ │ <h2>게시글 제목</h2>                              │ │
  │ │ 게시글 설명 1~2줄...                              │ │
  │ │ #tag1 #tag2                                      │ │
  │ └─────────────────────────────────────────────────┘ │
  │   ... (20개 반복)                                    │
  │   < 1 [2] 3 4 5 >                                    │
  └─────────────────────────────────────────────────────┘
  ```

### S-02: Article Detail

- **목적**: 게시글 본문 + 댓글 영역. 작성자에게 수정·삭제 액션 노출. 로그인 사용자에게 댓글 폼 노출.
- **상태**:
  - **default (비회원)**: 본문 + 댓글 목록만 (댓글 폼 미노출)
  - **default (가입 사용자, 비작성자)**: 본문 + 댓글 작성 폼 + 본인 댓글에만 수정/삭제 아이콘
  - **default (작성자)**: 본문 + 글 수정/삭제 버튼 + 댓글 폼 + 본인 댓글 수정/삭제
  - **댓글 인라인 편집**: 댓글 옆 [수정] → 편집 폼 → 저장
  - **삭제 모달**: [삭제] 클릭 시 확인 모달
  - **404**: "게시글을 찾을 수 없습니다"
- **F-ID 매핑**: F-02 (R-F-05, R-F-07, R-F-08) + F-03 (R-F-09, R-F-10, R-F-11, R-F-13)
- **주요 액션**: 글 [수정] → S-03 (편집 모드) / 글 [삭제] → 모달 → 확인 → DELETE / 댓글 폼 [등록] / 댓글 [수정] / 댓글 [삭제]
- **와이어프레임**:
  ```
  ┌─────────────────────────────────────────────────────┐
  │ Header                                               │
  ├─────────────────────────────────────────────────────┤
  │ <h1>게시글 제목</h1>                                  │
  │ [author] · 2026-05-21          [수정] [삭제]         │  ← 작성자만 표시
  │                                                       │
  │ <p>본문 단락 …</p>                                    │
  │                                                       │
  │ #tag1 #tag2                                           │
  │ ─────────────────────────────                         │
  │ <h2>댓글</h2>                                          │
  │ ┌─────────────────────────────────┐                  │
  │ │ [댓글 입력]                       │  ← 로그인 시      │
  │ │ [등록]                            │                  │
  │ └─────────────────────────────────┘                  │
  │ ┌─────────────────────────────────┐                  │
  │ │ [author] · 본문   [수정] [삭제]   │  ← 본인 댓글만    │
  │ │              ────────────────── │                  │

### S-03: Editor (글쓰기 / 수정)

- **목적**: 새 글 작성 (`/editor`) 또는 기존 글 수정 (`/editor/:slug`). 동일 컴포넌트.
- **상태**:
  - **default (새 글)**: 빈 폼
  - **default (수정)**: 기존 값 채워진 폼
  - **submitting**: 버튼 disabled + 스피너
  - **error**: 422 응답의 한글 에러 메시지 인라인 표시
- **F-ID 매핑**: F-02 (R-F-06, R-F-07)
- **주요 액션**: [공개] → POST/PUT → 성공 시 S-02로 이동
- **와이어프레임**:
  ```
  ┌─────────────────────────────────────────────────────┐

  │ [제목                                              ] │

  │ [본문                                               │
  │                                                     │
  │                                                   ] │
  │ [태그 (쉼표 구분, 선택)                            ] │
  │                                                       │

## 3. 디자인 시스템 / 토큰

본 토큰은 Tailwind CSS 3.x 솔루션을 통해 구현된다. 12-scaffolding/python.md §8(스타일링 솔루션)이 본 토큰을 참조한다.

> 본 프로젝트는 백엔드 학습이 1차 목표이므로 디자인 시스템은 *최소 필수*로 정의. Tailwind 기본 토큰을 그대로 활용하며, 본 절은 그 기본값을 명시적으로 채택한다는 선언.

### 3.1 Color

| 토큰 | 역할 | Tailwind 클래스 | 값 |
|---|---|---|---|
| `color/primary` | 주 액션 (CTA 버튼, 링크) | `text-blue-600` `bg-blue-600` | #2563eb |
| `color/primary-hover` | 호버 상태 | `hover:bg-blue-700` | #1d4ed8 |
| `color/secondary` | 보조 텍스트·테두리 | `text-gray-600` `border-gray-300` | #4b5563 / #d1d5db |
| `color/neutral-bg` | 페이지 배경 | `bg-white` | #ffffff |
| `color/neutral-text` | 본문 텍스트 | `text-gray-900` | #111827 |
| `color/danger` | 삭제·에러 메시지 | `text-red-600` `bg-red-50` | #dc2626 / #fef2f2 |

### 3.2 Typography

| 토큰 | 역할 | Tailwind 클래스 |
|---|---|---|
| `font/family` | 본문·UI 공통 | Tailwind 기본 `font-sans` (system-ui, -apple-system, Segoe UI, …) |
| `font/h1` | 페이지 제목 | `text-3xl font-bold` (1.875rem) |
| `font/h2` | 섹션 제목 | `text-2xl font-semibold` (1.5rem) |
| `font/body` | 본문 | `text-base` (1rem) |
| `font/small` | 메타·작성일 | `text-sm text-gray-500` (0.875rem) |

스케일 최소 3단(small·body·h2) 이상 만족.

### 3.3 Spacing

Tailwind 기본 4px 단위 스케일 채택. 본 프로젝트가 사용하는 단계:

| 토큰 | 역할 | Tailwind |
|---|---|---|
| `spacing/1` | 작은 간격 (아이콘-텍스트) | `p-1` (4px) |
| `spacing/2` | 폼 필드 간격 | `gap-2` (8px) |
| `spacing/4` | 카드 패딩, 섹션 간격 | `p-4` `mb-4` (16px) |
| `spacing/8` | 페이지 외곽 여백 | `p-8` (32px) |

스케일 최소 4단 만족.

### 3.4 Component primitives

| 컴포넌트 | 역할 | 상태 variant |
|---|---|---|
| `Button` | CTA · 보조 · 위험 | `default` (primary), `secondary`, `danger`, `disabled` (submitting 시 자동) |
| `Input` | 텍스트 입력, 이메일, 비밀번호 | `default`, `error` (422 응답 시 빨강 테두리), `disabled` |
| `Textarea` | 본문·댓글 입력 | `default`, `error`, `disabled` |
| `Card` | 게시글·댓글 카드 | `default`, `hover` (목록에서 마우스 오버 시 그림자) |
| `Tag` | 태그 칩 | `default` |
| `Modal` | 삭제 확인 모달 | `open`, `closed` |
| `Spinner` | 로딩 표시 | `default` |

> 컴포넌트 라이브러리(MUI/AntD 등)는 사용하지 않음 — Tailwind utility-first + JSX로 직접 구현. 학습 부담 최소화.

## 4. 접근성

- **키보드 네비게이션**: form 요소·버튼·링크 tab 순서 자연. Modal 진입 시 첫 버튼에 focus, ESC로 닫기.
- **시맨틱 HTML**: `<header>`, `<main>`, `<nav>`, `<article>`, `<button>` 사용. div+onclick 지양.
- **폼 라벨**: 모든 `<input>`에 `<label htmlFor>` 연결. placeholder만 사용하지 않음.
- **색상 대비**: Tailwind 기본 팔레트는 WCAG AA 충족. 위 §3.1 토큰값으로 본문/배경 대비 4.5:1 이상.
- **에러 메시지**: 시각적 강조(빨강) + 텍스트 동시 제공 (색맹 대응).
- **모바일 호환**: Tailwind 반응형 클래스(`sm:` `md:`)는 학습 컨텍스트에서 *최소 viewport meta + 기본 흐름 레이아웃*만 적용. 풀 반응형은 Out of Scope.

> 본 학습 컨텍스트에서 a11y는 *기본 시맨틱·라벨 수준*만 강제하고, 풀 WCAG 검사는 Out of Scope.

## 5. Open Questions

1. 모바일 반응형 — 햄버거 메뉴 등 별도 분기 추가할지. 본 MVP는 기본 흐름 레이아웃만.
2. 로딩 상태 표시 방식 — 전역 스피너 vs 각 액션 버튼 내부 스피너. S-02·S-03에서 결정 가능.
3. 댓글 페이지네이션 — 본 MVP는 한 글당 댓글 N개 전부 반환 (게시글당 평균 0~10건 가정). 댓글이 많아지면 페이지네이션 추가 검토.
4. 다크모드 / 다국어 — Out of Scope 명시 (RFP §3).
