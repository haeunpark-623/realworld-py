## 1. 페르소나

| 페르소나 | 역할 | 환경 / 컨텍스트 | 주요 목표 |
|---|---|---|---|
| 비회원 방문자 | 게시글·댓글 *읽기 전용* | 데스크톱·모바일 브라우저, 로그인 없음 | 게시글 목록을 훑어보고, 관심 글의 상세·댓글을 읽는다 |
| 가입 사용자 | 글·댓글 *작성자* | 데스크톱 브라우저(주), 로그인 상태 | 회원가입·로그인 후 자기 글·댓글을 쓰고 관리한다 |

> 관리자·모더레이터 페르소나는 Out of Scope (RFP §2 명시).

## 2. 사용자 여정 (큰 그림)

### 2.1 비회원 여정

```
홈 진입 → 게시글 목록 (최신순, 20개 페이지네이션)
       → 글 카드 클릭 → 게시글 상세 (본문 + 댓글 목록)
       → [회원가입 유도 배너] → 회원가입 폼 → 가입 사용자 여정으로 합류
```

### 2.2 가입 사용자 여정 (골든패스, DoD §3)

```
회원가입 → 로그인 (JWT 발급, 로컬 스토리지 저장)
       → 글쓰기 폼 → 제출 → 새 글 상세 페이지
       → 댓글 작성 → 댓글 목록에 추가
       → 댓글 수정 → 댓글 body 갱신
       → 글 수정 → 본문 갱신
       → 글 삭제 → 홈으로 리다이렉트 (hard delete, DB row 제거)
       → 로그아웃 (클라이언트 토큰 폐기)
```

위 골든패스 1회 통과(gstack `/qa`)가 본 프로젝트 DoD §3.

### 2.3 권한 분기 큰 그림

| 행동 | 비회원 | 가입 사용자 (작성자) | 가입 사용자 (타인) |
|---|---|---|---|
| 게시글 목록·상세 읽기 | ✅ | ✅ | ✅ |
| 게시글 작성 | ❌ (401) | ✅ | ✅ |
| 게시글 수정/삭제 | ❌ (401) | ✅ | ❌ (403) |
| 댓글 작성 | ❌ (401) | ✅ | ✅ |
| 댓글 수정 | ❌ (401) | ✅ (자기 댓글) | ❌ (403) |
| 댓글 삭제 | ❌ (401) | ✅ (자기 댓글) | ❌ (403) |

## 3. Use Case

### UC-01: 회원가입 (가입 사용자 진입)

- **주체**: 비회원 방문자
- **선행 조건**: 등록되지 않은 이메일·username 보유. 비밀번호 8자 이상.
- **트리거**: 회원가입 폼 진입 후 [가입] 버튼 클릭
- **본 흐름**
  1. username·email·password 입력
  2. 클라이언트 폼 유효성 검증 (필수·이메일 형식·비밀번호 길이)
  3. POST /api/users 호출 (RealWorld spec 준수)
  4. 서버: 중복 검사 → bcrypt 해시 → DB 저장 → JWT 발급
  5. 클라이언트: JWT 저장 + 홈으로 이동
- **대안 흐름 (실패)**
  - 이메일·username 중복 → 422 + 한글 에러 메시지
  - 형식 위반 → 400 + 한글 에러 메시지
  - 서버 내부 에러 → 500 + 일반 안내 (Sentry 등은 Out of Scope)
- **사후 조건**: 가입 사용자 상태로 전환, JWT 보유

### UC-02: 로그인

- **주체**: 가입 사용자 (비로그인 상태)
- **선행 조건**: UC-01 완료
- **트리거**: 로그인 폼 진입 후 [로그인] 버튼 클릭
- **본 흐름**
  1. email·password 입력 → 클라이언트 검증
  2. POST /api/users/login 호출
  3. 서버: 이메일로 사용자 조회 → bcrypt 비교 → 일치 시 JWT 발급
  4. 클라이언트: JWT 저장 + 홈으로 이동
- **대안 흐름 (실패)**
  - 이메일 미존재 → 422 + "이메일 또는 비밀번호가 올바르지 않습니다"
  - 비밀번호 불일치 → 동일 메시지 (어느 쪽이 틀렸는지 누설 금지, RFP §NFR-04 보안 정신)
- **사후 조건**: 로그인 상태(JWT 보유)

### UC-03: 게시글 목록 조회 (비회원·가입 사용자 공통)

- **주체**: 비회원 방문자 또는 가입 사용자
- **선행 조건**: 없음
- **트리거**: 홈 진입 또는 페이지네이션 이동
- **본 흐름**
  1. GET /api/articles?limit=20&offset=0
  2. 서버: 최신순 게시글 20개 반환 (총 개수 포함)
  3. 클라이언트: 카드 리스트 렌더링 + 페이지네이션 UI
- **대안 흐름 (실패)**
  - DB 연결 실패 → 503 + "잠시 후 다시 시도해 주세요"
- **사후 조건**: 목록 화면 표시. p95 < 200ms (RFP §NFR-02)

### UC-04: 게시글 상세 조회

- **주체**: 비회원 방문자 또는 가입 사용자
- **선행 조건**: UC-03에서 글 카드 클릭 또는 직접 URL 접근
- **트리거**: 글 카드 클릭 / URL 접근
- **본 흐름**
  1. GET /api/articles/{slug}
  2. GET /api/articles/{slug}/comments
  3. 클라이언트: 본문 + 댓글 영역 렌더링 (로그인 시 댓글 폼 노출)
- **대안 흐름 (실패)**
  - slug 미존재 → 404 페이지
  - 본문이 비정상적으로 큰 경우(악의적 입력) → 서버는 길이 제한으로 400 (NFR-04 XSS·DoS 방어)

### UC-05: 게시글 작성

- **주체**: 가입 사용자
- **선행 조건**: 로그인 완료 (JWT 유효)
- **트리거**: 글쓰기 메뉴 → 폼 입력 → [제출] 클릭
- **본 흐름**
  1. 제목·설명·본문·태그(선택) 입력
  2. POST /api/articles (Authorization: Token <JWT>)
  3. 서버: 인증 검증 → slug 생성 (title → kebab-case) → 충돌 시 *숫자 suffix 부여* (예: `my-post` 존재 시 `my-post-2`, 또 존재 시 `my-post-3`) → DB 저장
  4. 응답: 새 글 정보 (slug 포함)
  5. 클라이언트: /article/{slug} 로 이동
- **대안 흐름 (실패)**
  - JWT 만료/누락 → 401 + 로그인 페이지 이동
  - 제목·본문 누락 → 422 + 한글 에러 메시지
  - slug 충돌(같은 제목 다수) → 서버에서 자동 숫자 suffix 부여로 회피 (에러 아님)

### UC-06: 게시글 수정

- **주체**: 가입 사용자 (해당 글 *작성자*)
- **선행 조건**: 본인이 작성한 글 존재, 로그인 상태
- **트리거**: 상세 페이지 → [수정] 버튼
- **본 흐름**
  1. 기존 값으로 폼 채움
  2. 수정 후 PUT /api/articles/{slug}
  3. 서버: 인증·작성자 검증 → 필드 갱신 → 응답
  4. 클라이언트: 상세 페이지 갱신
- **대안 흐름 (실패)**
  - 타인의 글 수정 시도 → 403 + "권한이 없습니다"
  - 비로그인 → 401

### UC-07: 게시글 삭제

- **주체**: 가입 사용자 (해당 글 *작성자*)
- **선행 조건**: 본인이 작성한 글 존재, 로그인 상태
- **트리거**: 상세 페이지 → [삭제] 버튼 → 확인 모달 [예]
- **본 흐름**
  1. DELETE /api/articles/{slug}
  2. 서버: 인증·작성자 검증 → **hard delete** (DB row 영구 제거. 게이트 B v0.2 결정). 댓글은 FK CASCADE로 함께 삭제 (세부는 게이트 C `08-lld-module-spec`에서 확정)
  3. 클라이언트: 홈으로 리다이렉트
- **대안 흐름 (실패)**
  - 타인의 글 삭제 시도 → 403
  - 동시 삭제(이미 삭제된 글) → 404 + "이미 삭제된 글입니다"

### UC-08: 댓글 작성

- **주체**: 가입 사용자
- **선행 조건**: 게시글 상세 페이지, 로그인 상태
- **트리거**: 댓글 폼 입력 + [등록]
- **본 흐름**
  1. POST /api/articles/{slug}/comments (body: { comment: { body } })
  2. 서버: 인증 검증 → 댓글 저장 → 응답
  3. 클라이언트: 댓글 목록 상단에 추가
- **대안 흐름 (실패)**
  - 비로그인 → 401
  - 본문 비어있음 → 422
  - 글 미존재 → 404

### UC-09: 댓글 삭제

- **주체**: 가입 사용자 (해당 댓글 *작성자*)
- **선행 조건**: 본인이 작성한 댓글 존재, 로그인 상태
- **트리거**: 댓글 옆 [삭제] 아이콘 → 확인
- **본 흐름**
  1. DELETE /api/articles/{slug}/comments/{id}
  2. 서버: 인증·작성자 검증 → 삭제
  3. 클라이언트: 댓글 목록에서 제거
- **대안 흐름 (실패)**
  - 타인의 댓글 삭제 시도 → 403
  - 이미 삭제된 댓글 → 404

### UC-10: 로그아웃

- **주체**: 가입 사용자
- **선행 조건**: 로그인 상태
- **트리거**: 헤더의 [로그아웃] 클릭
- **본 흐름**
  1. 클라이언트: 로컬 스토리지 JWT 폐기
  2. 홈으로 이동
- **대안 흐름**: 서버 측 토큰 무효화 미적용(JWT stateless, RFP §F-01d). Out of Scope로 명시.

### UC-11: 내 프로필 조회 (내 글 목록)

- **주체**: 가입 사용자
- **선행 조건**: 로그인 상태
- **트리거**: 헤더의 [내 프로필] 클릭
- **본 흐름**
  1. GET /api/articles?author={username}
  2. 본인 작성 글 목록 렌더링
- **대안 흐름 (실패)**
  - 비로그인 시 헤더에 [내 프로필] 노출 안 함

### UC-12: 댓글 수정 (작성자 한정)

- **주체**: 가입 사용자 (해당 댓글 *작성자*)
- **선행 조건**: 본인이 작성한 댓글 존재, 로그인 상태
- **트리거**: 댓글 옆 [수정] 아이콘 → 인라인 편집 폼 → [저장]
- **본 흐름**
  1. 기존 body로 편집 폼 채움
  2. PUT /api/articles/{slug}/comments/{id} (body: { comment: { body } })
  3. 서버: 인증·작성자 검증 → body 갱신 → 응답 (수정 댓글 객체)
  4. 클라이언트: 해당 댓글 위치에서 즉시 갱신
- **대안 흐름 (실패)**
  - 타인의 댓글 수정 시도 → 403
  - body 빈 문자열 → 422 + 한글 에러
  - 이미 삭제된 댓글 수정 시도 → 404 + "이미 삭제된 댓글입니다"
  - 비로그인 → 401

## 4. 비기능 시나리오

| 시나리오 | 조건 | 기대 동작 | 관련 NFR |
|---|---|---|---|
| 게시글 100건 시드 후 목록 응답 | 게시글 100건 + DB seeded | API p95 < 200ms | NFR-02 |
| 게시글 목록 첫 페인트 | dev 서버, 게시글 100건 | FCP < 1.5s (로컬 측정) | NFR-02 |
| 비밀번호 저장 | 회원가입 직후 DB 조회 | password 컬럼이 bcrypt 해시 (`$2b$...`)로 시작, 평문 미포함 | NFR-04 |
| JWT secret 출처 | 서버 부팅 | 환경변수에서 로드, 코드/저장소에 미커밋 | NFR-04 |
| XSS 입력 | 본문 `<script>alert(1)</script>` 작성 | 서버 응답에서 escape되어 alert 미발화 | NFR-04 |
| CSRF (서버 렌더링 채택 시) | 외부 사이트에서 POST 요청 | 토큰 미동봉으로 거부 (403) | NFR-04 |
| 모바일 브라우저 호환 | Chrome Android 최신 | 폼·목록·상세 화면 깨짐 없이 렌더링 | NFR-03 |
| 폼 유효성 메시지 | 빈 필수 입력 제출 | 한글 에러 메시지 노출 | NFR-03 |
| fresh checkout 부팅 | `LOCAL.md` §3 명령 실행 | dev 서버 기동까지 ≤ 5분 | NFR-06 |
| 단위 테스트 커버리지 | pytest --cov, 핵심 비즈니스 로직(인증·권한·CRUD) | ≥ 80% (ADR-0015) | NFR-05 |
